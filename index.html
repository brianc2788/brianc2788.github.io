Shmup Update - Jan 10, 2023

I've recently become quite interested in developing my own game. It's something I've wanted to do for a long time, but I wanted to do it right. What I mean by that is, basically, that I wanted to create my own engine to run my game on. I know it's very normal practice to license a game engine, like Unreal or Unity, but I wanted to create my game from scratch as much as possible. This is why I've chosen to create the engine from scratch with a cross-platform library (like SDL2), which will also be the basis for a 2-D game engine, and I plan to move on to using OpenGL for a 3-D engine in the future. Because OpenGL can't really do anything beyond graphics, I'll continue to use SDL to handle things like windowing, events, input, etc.

I'm also working on a simple, ping-like game that will have multiplayer. I've done network programming before, but I'm very excited to get a chance to try creating a game. I think a game will have its own, unique set of challenges. Games are very dynamic applications that need low-latency network solutions to facilitate smooth gameplay and visuals.

I wanted to also give a shout-out to LazyFoo Productions; they have some excellent tutorials for using SDL2 & OpenGL that helped me get my start into using these libraries.

Building SFML on Linux

Hey, all.
Just wanted to share a couple thoughts after trying out SFML, a library for developing cross-platform, graphical applications. I was highly interested in using this library for games and GUIs and wanted to see how it compares to another popular library, SDL2. I followed a tutorial on the official SFML site, which demonstrated how to quickly create a window, texture, some text, and render it to the screen. My first impression is that this is much easier to use than SDL2. I can't say a whole lot, since I haven't done much with it yet, but I'm worried that the ease-of-use may have trade-offs with granularity and control over certain aspects of the windowing/rendering subsystems. At this point, I'm planning to continue to use SDL2 to develop a couple games/demos (including the vert. shooter from past posts), but I'm seriously considering a game or GUI with SFML afterwards.

I'd like to take this as an oppertunity to say how pleased I am with Linux as a development environment. Many Linux distributions will have development libraries available in their package repositories, but even if yours doesn't, it's still easier to install third-party libraries than it is on Windows for Visual Studio (in my humble opinion). Visual Studio and Visual C are notorious, especially for new programmers, for being difficult to set options for compilation & linking. For me, using SFML was as simple as installing the libraries via a package manager, adding the include directives to the source code, and passing arguments to the compiler/linker (I use the Gnu Compiler Collection & Make).

Check out SFML on their official website:
www.sfml-dev.org

2D Shmup Demo

Every once in awhile I get an itch to make a video game. These screenshots show a couple game demos intended to become a small, vertically-scrolling, arcade-style shooter game. Though it's not the first of its kind from me, it's my most recent attempt. I started just to try out Python's pygame mod which uses bindings for the SDL2 development libraries. The part of game development I find most difficult is creating/acquiring assets. Feeling slightly more ambitious, I moved on to re-use these assets and rewrite my little game engine in C++.
Currently, I'm still playing with the code at times. I just want to refamiliarize myself with SDL2 and a couple of its supporting libraries (image, ttf). I probably won't continue on this particular demo for much longer, but after a couple more exercises with graphics and GUIs, I might come back to it.
If the 8-bit esque aesthetics are off-putting to you, feel free to create something better and donate it to me! (Lol<3). I was somewhat inspired by the original Gameboy's presentation; so much so that I also considered a black-and-puke-green color palette! For those who are interested in game development and also struggling with art, there are alternatives, such as buying assets on the Unity game engine's online market. Furthermore, for aspiring game developers, there's never been a more accessible time. Using Unity as en example again, they have their own, entirely unique scripting engine that someone could theoretically learn and create a full-on, triple-A video game title from scratch with (don't try this at home lol).

Thanks for checking it out and feel free to contact me.

Book Endorsements

I read them and you should too!

Just wanted to share a few of the books that currently have my attention. I absolutely love O'Reilly & No Starch books.
